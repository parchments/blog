git-introduction

一、历史与发展

    Linus -> Linux -> no svn -> bitKeeper(收费) -> Git

 

二、集中式与分布式的区别

1、文件保存方式不同

    1.1、SVN记录的是哪个文件修改了，记录了差异

    1.2、把任意时间点的快照记录了，并且每个版本之间还会比较。（快照：快照表示某个时间点对应的代码长什么样）

 

2、工作方式以及出问题后解决办法

    2.1、SVN是集中化的版本控制系统，用同一个服务器，一旦出问题，所有人都不能提交更新。历史记录存放位置单一，出现问题可能面临版本丢失的问题

    2.2、Git是分布式存储管理，客户端存的是原始代码的完整镜像，存储到本地，也就是说每一个客户端的代码仓库都是一样的，任何部分代码出现问题，可以使用其他客户端的代码惊醒修复

 

三、Git本地仓库

1、下载与安装

下载地址：https://git-scm.com/

安装完之后任意一个文件夹右键会出现

新建文件夹并在当前文件夹下面右键打开

 

2、git第一个命令

初始化本地仓库 git init，发现多了一个.git的底层文件，不能动，以后的版本信息等等都会存在这里。

 

四、远程仓库

1、注册账号: https://github.com/, 用户账号以及邮箱，邮箱验证码等等

2、创建项目

登录后进入首页

远程分支已经创建完毕（项目文件夹）

 

五、本地与远程关联

1、SSH配置（与远程关联的配置）

    SSH配置：ssh-Keygen

    配置完之后能得知ssh的文件位置，并且复制ssh里面的key

    然后把复制的key粘贴在github-我的-setting-SSH and GPG keys-Add new中去

    远程与本地关联成功

 

2、本地环境配置：

    git config –global user.name 'xxx'

    git config –global user.email 123456789@qq.com

    这里可以知道谁对项目进行了操作

3、指令

3.1、初始化本地仓库：git init

3.2、添加源仓库：git remote add origin yourgithub（切换到user-ssh地址）

3.3、把远程仓库的内容下载下来：git pull origin master

3.4、检查绑定情况，查看远程仓库：git remote -v

3.5、查看当前文件的状态： git status

3.6、提交到暂存区： git add <pathspec>(如果单个文件就像index.html，如果多个文件就用“ . ”点的方式)

    暂存区用处：有修改但是没写完就不能提交到本地仓库，提交到本地仓库都叫一个版本，万一有bug提交后有人合并就会出问题。暂存区会被最新的代码覆盖之前的代码

    放到暂存区：

3.7、提交到本地仓库：git commit -m ' 提交信息的简介 '

3.8、提交内容到远程仓库： git push <remote> <branch>

3.9、撤销：git checkout – index.html

    把自己的工作区恢复至最近的一个暂存区的内容情况

3.10、版本号回退：

  • 查看日志，每次提交了什么： git log 

    其中commit 后面的b7dc85是版本号，可以进行版本号回退

  • 回退一个完整的版本：–hard，回退到上一个完整的版本

git reset –hard HEAD~1

git reset –hard HEAD~2(回退两个版本)

 

  • mixed回退到暂存区添加之前的状态，即add状态

 

  • soft之后要么版本回退要么提交，退到commit状态

  • 回退到指定版本：git reflog

    git reset –hard b7dc854

 

3.11、其中origin可以替换，只是远程服务器的一个名字而已，

    git remote rename origin dev

    git remote

    返回dev

    获取github项目的地址

3.12、克隆仓库：git clone <repository>

3.13、远程仓库重命名：git remote rename oldName newName(这步可改可不改)

3.14、添加源仓库地址为远程仓库： git remote add <name> <remote>

3.15、查看文件夹： dir

差异比较：

3.16、比较工作区与暂存区： git diff

3.17、比较暂存区与本地版本库中最近一次commit 的内容： git diff –cached

3.18、比较工作区与本地版本库中最近一次commit的内容 ：git diff HEAD

3.19、比较两个commit之间的差异： git diff <commit-id> <commit-id>

 

基础工作流

3.20、查看分支： git branch

3.21、创建新分支： git branch <name>，新建完后要切换分支，然后add，commit，push，最后在github才有显示

3.22、创建并切换分支： git checkout -b <name>

3.23、删除本地分支： git branch -d <name>

3.24、删除远程分支： git push -d <origin> <branch>

3.25、切换分支： git checkout <name>

3.26、删除某个文件： git rm -r –cached target  删除target文件夹后commit，push到master

master是装着没有bug的代码，dev分支是跟master一样的代码，但是允许报错

3.27、合并分支：git merge <branch>

3.28、把本地master分支推送到远程的master分支上：

git push origin master:master

git push origin master:dev
